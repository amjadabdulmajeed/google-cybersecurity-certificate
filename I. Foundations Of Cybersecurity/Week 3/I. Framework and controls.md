## Security Frameworks and Controls

### Security Framework
Guidelines for building security plans
= proven blueprint for security
Purpose:
→ Protect PII
→ Secure financial information
→ Identify weaknesses
→ Manage organizational risk
→ Align security with business goals

### Security Lifecycle
Constantly evolving set of policies + standards
Never "done" — continuous cycle

### 4 Core Framework Components
1. Identify + document security goals
   → WHERE ARE WE NOW?
2. Set guidelines to achieve goals
   → WHERE DO WE NEED TO BE?
3. Implement strong security processes
   → HOW DO WE GET THERE?
4. Monitor + communicate results
   → ARE WE STAYING THERE?
   → back to step 1 (continuous)

### Security Controls
Safeguards reducing specific security risks
3 types:
→ Technical: firewalls, MFA, SIEM, encryption
→ Administrative: policies, training, procedures
→ Physical: keycards, cameras, locks

### The Complete Picture:
CIA Triad → WHAT we protect
CISSP Domains → WHO protects it
Frameworks → HOW protection is structured
Controls → SPECIFIC TOOLS used
Together = complete security posture
